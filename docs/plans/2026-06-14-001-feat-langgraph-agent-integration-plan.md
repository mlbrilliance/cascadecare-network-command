# feat: Integrate forensic-self-exam-agent LangGraph conversion into production architecture

## Summary

The `forensic-self-exam-agent-langgraph` scaffold (LangGraph `StateGraph`, 14 green tests) was
authored on `feat/langgraph-agent`. This plan wires it into the production architecture using the
official `uipath-langchain-python` path (Option B): `langgraph.json` manifest → `uipath init`
codegen → `entry-points.json` + `bindings.json`, matching the deploy pattern of the four existing
coded agents (`uip codedagent pack / publish`). It also adds the project scaffold files
(`project.uiproj`, `AGENTS.md`, `uv.lock`) that the UiPath CLI requires, and updates `CLAUDE.md`
to record that LangGraph is permitted at the agent-SDK layer.

---

## Problem Frame

The scaffold exists and tests pass, but the agent is not deployable or architecturally complete:

- `uipath.json` with `"type": "agent"` was used instead of `langgraph.json` (the `uipath-langchain`
  path is different from the standard coded-agent path and requires a separate manifest)
- `entry-points.json` and `bindings.json` are absent — generated by `uipath init` reading
  `langgraph.json`, required for `uip codedagent pack`
- `project.uiproj` is absent — required by the UiPath CLI to recognize the project
- `uipath-langchain` is not in the agent's `pyproject.toml`
- `uv.lock` is absent — existing coded agents commit a lockfile for reproducible builds
- `CLAUDE.md` still says "no LangGraph, no Python harness" — now incorrect at the agent layer

**Scope boundary:** Maestro Case remains the orchestrator. This plan converts one agent's SDK
implementation from `uipath-python` (function pattern) to `uipath-langchain` (LangGraph graph
pattern). The Maestro Case canvas, the five reversals, HITL gates, and all other agents are
unchanged.

---

## Requirements

1. Agent is deployable via `uip codedagent pack` and `uip codedagent publish` on the same pipeline
   as the other four coded agents.
2. I/O schema (three inputs: `clearflow_indicators`, `nimbus_indicators`, `clearflow_self_victim`;
   four outputs: `route_to`, `clearflow_vector_status`, `rationale`, `error_type`/`error_message`)
   is preserved and reflected in the generated `entry-points.json`.
3. All 14 existing tests remain green.
4. `uv run pytest` (full suite, 677+ tests) passes after dependency changes.
5. `CLAUDE.md` accurately describes what LangGraph is permitted for.
6. The new agent follows the same file-layout conventions as
   `agents/claim-flow-anomaly-detector/` and `agents/forensic-self-exam-agent/`.

---

## Key Technical Decisions

**KTD-1: `langgraph.json` as the sole manifest; `uipath.json` removed.**
The `uipath-langchain` SDK reads `langgraph.json` to discover the graph entrypoint and generates
`entry-points.json` + `bindings.json` via `uipath init`. The existing `uipath.json` (which carried
a hand-written schema) is replaced entirely. The generated `entry-points.json` becomes the
authoritative schema — do not hand-edit it after generation.

**KTD-2: `uipath-langchain` added to agent `pyproject.toml`; `langgraph` and `langchain-core`
remain pinned there.**
`uipath-langchain` wraps the `uipath-python` SDK with LangGraph support. It is an agent-local
dependency (not added to root `pyproject.toml`, which is build-time tooling only). The
`[tool.uipath] type = "agent"` section in the existing pyproject.toml is removed — it is not used
by the `uipath-langchain` path.

**KTD-3: `project.uiproj` uses `"ProjectType": "Agent"`.**
Existing coded-function agents use `"ProjectType": "Function"`. The LangGraph path registers as
an `Agent` project type, matching the UiPath Agent Builder naming convention.

**KTD-4: `uv.lock` is committed.**
All five coded agents commit a `uv.lock`. Reproducible builds are a project convention.
Generate once with `uv lock` in the agent directory after finalizing `pyproject.toml`.

## High-Level Technical Design

### Manifest and codegen pipeline

```
agents/forensic-self-exam-agent-langgraph/
  langgraph.json          ← NEW: points to agent:graph
  agent.py                ← existing StateGraph (unchanged)
  pyproject.toml          ← updated: add uipath-langchain, remove [tool.uipath]
  uv.lock                 ← NEW: committed lockfile
  entry-points.json       ← GENERATED by `uipath init` (replaces hand-authored schema)
  bindings.json           ← GENERATED by `uipath init`
  project.uiproj          ← NEW: ProjectType = Agent
  AGENTS.md               ← NEW: agent-specific code patterns ref (mirrors other agents)
  uipath.json             ← DELETED
  tests/                  ← unchanged (14 tests green)
```

### How `uipath init` reads `langgraph.json`

```
langgraph.json: {"graphs": {"agent": "agent:graph"}}
       │
       ▼
uipath init (uipath-langchain CLI)
       │ introspects ForensicState TypedDict
       │ maps input keys → JSON Schema
       │ maps output keys → JSON Schema
       ▼
entry-points.json   (type: "agent", input/output from ForensicState)
bindings.json       (version: "2.0", resources: [])
```

### Deploy pipeline (unchanged from other coded agents)

```
uv lock                                    # (once after pyproject.toml change)
uipath init                                # generates entry-points.json + bindings.json
uip codedagent pack   -n forensic-self-exam-agent-langgraph -v <ver>
uip codedagent publish -n forensic-self-exam-agent-langgraph -v <ver>
```

## Implementation Units

### U1. Replace uipath.json with langgraph.json

**Goal:** Swap the hand-written `uipath.json` manifest for the `langgraph.json` required by the
`uipath-langchain` SDK.

**Requirements:** R1 (deployable), KTD-1.

**Dependencies:** none.

**Files:**
- `agents/forensic-self-exam-agent-langgraph/langgraph.json` — create
- `agents/forensic-self-exam-agent-langgraph/uipath.json` — delete

**Approach:** `langgraph.json` must contain `{"graphs": {"agent": "agent:graph"}}` — the key
`"agent"` is the graph name the SDK uses, and `"agent:graph"` points to the `graph` variable
exported at module level in `agent.py`. No other fields are required.

**Test scenarios:**
- Happy path: `langgraph.json` is valid JSON, contains `graphs.agent` key pointing to `agent:graph`
- Structural: `uipath.json` no longer exists in the directory

**Verification:** `cat agents/forensic-self-exam-agent-langgraph/langgraph.json` parses correctly;
`ls agents/forensic-self-exam-agent-langgraph/uipath.json` returns non-zero.

### U3. Generate entry-points.json and bindings.json via uipath init

**Goal:** Produce the UiPath CLI-required codegen artifacts from the LangGraph manifest.

**Requirements:** R1, R2, KTD-1.

**Dependencies:** U1, U2.

**Files:**
- `agents/forensic-self-exam-agent-langgraph/entry-points.json` — generated (commit)
- `agents/forensic-self-exam-agent-langgraph/bindings.json` — generated (commit)

**Approach:** From within `agents/forensic-self-exam-agent-langgraph/`, run:

```

```

The `uipath-langchain` package installs a `uipath` CLI entry point. `uipath init` reads
`langgraph.json`, introspects `ForensicState` (the `StateGraph`'s type annotation), and writes
`entry-points.json` and `bindings.json`. If the generated `entry-points.json` does not reflect
the expected I/O schema (see R2), check that `ForensicState` TypedDict annotations are fully
explicit — the SDK introspects type hints, not docstrings.

**Implementation note:** If `uipath init` requires the graph to be importable, ensure `langgraph`
and `langchain-core` are installed in the local venv first (`uv sync`). Do not hand-edit the
generated files after generation.

**Test scenarios:**
- `entry-points.json` is valid JSON with `$schema` field matching the UiPath entry-point schema
- `entry-points.json` `entryPoints[0]` contains `input.properties` with all three expected input
  keys (`clearflow_indicators`, `nimbus_indicators`, `clearflow_self_victim`)

  `clearflow_vector_status`, `rationale`

**Verification:** Generated files are present; `jq '.entryPoints[0].input.properties | keys'
